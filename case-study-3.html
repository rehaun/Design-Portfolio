Redesign Case Study

What’s included:

Context

Classeh is a School Management System. The dashboard was outdated and identical for all users. We redesigned it to create three tailored versions for parents, students, and teachers, focusing on their specific needs. As a multiple roles platform, let’s see how we turned a single UI between all roles into several dedicated pages for each role according to their needs: parents, students, and teachers.

BEFORE & AFTER

🕵🏼‍♂️ Research

✨ Ideation

🎨 Visual Design

Problem

The old dashboard was the same for all users, making it hard for them to find what they needed quickly. It lacked a user-friendly design and was visually unappealing.

Module tiles — main page of the platform in all user roles

Discovery

Process

Assumptions

I had a relative knowledge, but I was not sure what exactly the users needed. My assumption: students want to do a task assigned to them, such as doing homework or taking a test. Teachers come to ask to do something and do the next steps, such as creating assignments and tests and then correcting them. Parents are more likely to find out about the child’s academic status and talk to the teacher.

User Proxy Interviews

To evaluate the assumption, I talked about this with the support staff who have direct and indirect contact with the users and I checked the analysis tools and saw that it is almost as I thought.

Design

Rapid Prototype

We decided it should be a sidebar. I made a quick version to get feedback, because I already learned that it is better to share it as soon as possible and not spend time on it in isolation.

The first implementation of the idea

Feedback & Iteration

In our presentation meeting with the CTO, the sidebar was rejected for some reasons and the shortcuts were approved. Now, instead of the sidebar, I polished the visual of the tiles in the first step.

Tiles were visually improved

Apply the Categories

And after consulting with my colleagues, we applied the classification in this way: inside the previous dashboard page (which we called Hub now), I separated the module categories according to their type and named them.

Result

Although we did not gather direct feedback from users, the support team reported a significant improvement in user satisfaction. Users frequently called in to express their gratitude for the changes. Additionally, the dashboard’s performance improved by 10%, as measured through Microsoft Clarity, indicating a more efficient and responsive user experience.
